## 📡 รายงานจากสายข่าว

สามวันหลังจากที่เราทุกคนรายงานตัวเข้าหน่วยและยื่นแฟ้มประวัติส่วนตัว — Push `my-introduction.md` ขึ้น GitHub กันไปเรียบร้อย...

สายข่าวแจ้งเข้ามาเมื่อคืน: **Interpol** ตามรอยกลับมาถึง GitHub Repository ที่ใช้รับสมัครสมาชิก ไฟล์ `my-introduction.md` ของทุกนายยังเปิดโล่งบนคลาวด์ — และมันพาพวกเขามาถึงรัง พวกเขากำลังรวบรวมหลักฐาน และมีแผนบุกตรวจค้นรังซุ่มภายใน 72 ชั่วโมง

บอสใหญ่เรียกประชุมฉุกเฉิน ใบหน้าเย็นชา เสียงต่ำ:

> _"ถ้าพวกมันตรวจสอบประวัติย้อนหลังได้ พวกเราเสร็จ ทางรอดทางเดียวคือ — เราต้องสร้าง **โลกคู่ขนาน** ขึ้นมา ประวัติที่สะอาด ไร้มลทิน และถ้าหลักฐานมัดตัวหลุดขึ้นไปแล้ว ก็ต้องถอนพิษมันออกให้ได้"_

ภารกิจวันนี้ไม่มีที่สำหรับความผิดพลาด

## ขั้นเตรียมการ: ตั้งฐานปฏิบัติการก่อนลงมือ

ก่อนจะสร้างอีลิบาย คุณต้องมีฐานปฏิบัติการเป็นของตัวเองก่อน — วิธีเดียวกับตอนสมัครเข้าแก็งครั้งแรก (อ้างอิงขั้นตอนจาก [`Workshop-Class01/instruction.md`](../Workshop-Class01/instruction.md))

**ขั้นที่ 1:** Fork repo นี้ให้เป็นของตัวเอง

กด Fork ที่มุมขวาบนของหน้า GitHub repo เพื่อคัดลอก repo มาไว้ใน account ของตัวเอง

**ขั้นที่ 2:** Clone แล้ว cd เข้าไปในฐานปฏิบัติการ

```bash
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>
```

**ขั้นที่ 3:** ตั้งค่าตัวตนใน repo (local config)

```bash
git config --local user.name "Full name"
git config --local user.email "Your Email"
```

พร้อมแล้ว — ตอนนี้เข้าไปอยู่ในฐานปฏิบัติการของตัวเองเรียบร้อย เริ่มภารกิจต่อไปได้

---

## ภารกิจที่ 1: สร้างโลกคู่ขนานหลอก Interpol

ลองนึกภาพว่าชีวิตคุณมีสองเส้นทาง: เส้นทาง **`main`** คือตัวตนนักฆ่าที่แท้จริง และเส้นทาง **`alibi-timeline`** คือตัวตนที่คุณสร้างขึ้นมาเพื่อหลอกตำรวจ — ชายผู้ป่วยหนักนอนอยู่บนเตียงโรงพยาบาล ไม่มีทางออกไปก่อเหตุได้ คุณสลับไปมาระหว่างสองตัวตนได้ตามต้องการ และเมื่อ Interpol เข้ามาตรวจ ก็ชี้ให้พวกเขาเห็นแค่ตัวตนที่ต้องการให้เห็น

### 🔫 ปฏิบัติการ

**ขั้นที่ 1:** สร้างโลกคู่ขนานขึ้นมา

```bash
git switch -c alibi-timeline
```

คุณเพิ่งแยกตัวเองออกจากสายหลักแล้ว ตอนนี้อยู่บนเส้นทาง `alibi-timeline`

**ขั้นที่ 2:** เปิดไฟล์ `hit_list.txt` แล้วลบรายชื่อเป้าหมายทั้งหมดออก แทนที่ด้วยตารางนัดแพทย์ปลอม เช่น:

```
ตารางนัดหมายแพทย์ — โรงพยาบาลกรุงเทพ
- 08:00  เจาะเลือดตรวจไข้หวัดใหญ่สายพันธุ์ H3N2
- 10:30  เอกซเรย์ปอด (ผลยืนยันปอดอักเสบเล็กน้อย)
- 14:00  รับน้ำเกลือและยา — นอนพักในวอร์ด C
```

**ขั้นที่ 3:** บันทึกหลักฐาน

```bash
git add hit_list.txt
git commit -m "chore: บันทึกตารางแพทย์และอาการไข้หวัดใหญ่"
```

อีลิบายถูกบันทึกลงในประวัติแล้ว เมื่อ Interpol ขอตรวจสอบ พวกเขาจะเห็นแค่คนป่วยที่ไม่มีทางออกไปก่อเหตุได้

## ภารกิจที่ 2: สลับกลับโลกจริง — และออกจากกับดัก Detached HEAD

อีลิบายพร้อมแล้ว — ถึงเวลาสลับกลับตัวตนจริง

### 🔫 ปฏิบัติการ

**ขั้นที่ 1:** สลับกลับสายหลัก

```bash
git switch main
```

**ขั้นที่ 2:** สมาคมส่งข้อความมาทางช่องทางลับ — มีสคริปต์อัปเดตระบบรักษาความปลอดภัยใน repo แล้ว ให้รันก่อนเริ่มภารกิจต่อไป

```bash
bash system-update.sh
```

พอรันปุ๊บ terminal แสดงสัญญาณแปลก:

```
HEAD detached at a1b2c3d
```

สคริปต์พาคุณย้อนกลับไปในประวัติศาสตร์โดยไม่รู้ตัว — ตอนนี้คุณลอยอยู่กลางอากาศ ไม่มีที่ยึด

**ขั้นที่ 3:** ตรวจสอบว่าตัวเองอยู่ที่ไหน

```bash
git status
git log --all --oneline -5
```

**ขั้นที่ 4:** ดึงตัวเองกลับมาที่ยอดสาย main

```bash
git switch main
```

คุณปลอดภัยแล้ว — กลับมาอยู่บนสายหลักเรียบร้อย

---

## ภารกิจที่ 3: กับดักคำสารภาพบาป และการทำลายหลักฐาน

กลับมาบน `main` แล้วลองดู log:

```bash
git log --oneline -5
```

มีบางอย่างซ่อนอยู่กลาง history:

```
chore: อัปเดตระบบรักษาความปลอดภัยเรียบร้อย
confession: บันทึกการปฏิบัติการ               ← หลักฐานมัดตัว
ops: ปรับแฟ้มปฏิบัติการ...
```

สคริปต์แอบฝัง confession ไว้กลาง log แล้วมี commit ปกติทับหน้าอยู่ ตอนนี้ต้องเลือกว่าจะถอนพิษยังไง

---

### มาตรการที่ 1: ลบร่องรอยก่อนหลุดออก (ยังไม่ได้ Push)

```bash
git log --oneline -5
```

จดจำ hash ของ Commit ที่อยู่ **ก่อนหน้า** confession แล้วทุบทิ้ง — commit ที่อยู่เหนือ confession จะหายไปด้วย:

```bash
git reset --hard <commit-hash-ก่อนหน้า-confession>
```

Commit สารภาพบาปหายไปจากโลกนี้แล้ว ไม่มีร่องรอย Interpol ไม่มีวันพิสูจน์ได้

---

### มาตรการที่ 2: ถอนพิษหลักฐานที่หลุดออกไปแล้ว (Push ไปแล้ว)

สมมติว่าสายลับมือใหม่กด Push ไปก่อนจะทันสังเกต วิธีที่ถูกต้องคือสร้าง Commit ใหม่ที่ **ยกเลิกผลของ confession โดยไม่แตะ commit อื่น**:

```bash
git log --oneline -5
# หา hash ของ confession

git revert <commit-hash-confession>
# Editor จะเปิดขึ้นมาให้แก้ข้อความ Commit
# บันทึกแล้วปิด

git push origin main
```

ใน `git log` จะมีสองบรรทัด:

```
a9f3c1e  Revert "confession: บันทึกการปฏิบัติการ"   ← ตัวถอนพิษ
b7e2d4a  confession: บันทึกการปฏิบัติการ             ← ของเก่าที่ถูกยกเลิกแล้ว
```

ประวัติยังอยู่ครบ แต่ผลลัพธ์สุดท้ายสะอาด — เหมือนสายลับที่ยอมรับความผิดพลาดอย่างเป็นทางการ แล้วแก้ไขมันอย่างมีระเบียบวินัย

---

## ⚠️ กฎเหล็กของ Sicario-script

> **ห้ามใช้ `git reset --hard` กับ Commit ที่ Push ขึ้น Remote ไปแล้วเด็ดขาด**  
> ประวัติของทุกคนในทีมจะกระจัดกระจาย สมาชิกคนอื่นจะสับสน และ Interpol จะรู้ทันทีว่ามีคนพยายามซ่อนอะไรบางอย่าง  
> ทางรอดเดียวในสถานการณ์นั้นคือ `git revert` — สุภาพ เป็นระเบียบ และทีมไม่เดือดร้อน
