# 🔴 คลาส 3: แผนซ้อนแผน — สร้างหลักฐานเท็จสะกดรอยตำรวจ

> **สถานะ:** ลับสุดยอด | ระดับภัยคุกคาม: สูงสุด  
> **จาก:** บอสใหญ่ Sicario-script  
> **ถึง:** ปฏิบัติการทุกนาย

---

## 📡 รายงานจากสายข่าว

สามวันหลังจากที่เราทุกคนรายงานตัวเข้าหน่วยและยื่นแฟ้มประวัติส่วนตัว...

สายข่าวแจ้งเข้ามาเมื่อคืน: **Interpol** ตรวจพบคราบเลือดดิจิทัลที่สมาคมทิ้งไว้ในระหว่างปฏิบัติการ พวกเขากำลังรวบรวมหลักฐาน และมีแผนบุกตรวจค้นรังซุ่มภายใน 72 ชั่วโมง

บอสใหญ่เรียกประชุมฉุกเฉิน ใบหน้าเย็นชา เสียงต่ำ:

> _"ถ้าพวกมันตรวจสอบประวัติย้อนหลังได้ พวกเราเสร็จ ทางรอดทางเดียวคือ — เราต้องสร้าง **โลกคู่ขนาน** ขึ้นมา ประวัติที่สะอาด ไร้มลทิน และถ้าหลักฐานมัดตัวหลุดขึ้นไปแล้ว ก็ต้องถอนพิษมันออกให้ได้"_

ภารกิจวันนี้ไม่มีที่สำหรับความผิดพลาด

---

## 🎯 สิ่งที่จะได้เรียนรู้จากปฏิบัติการนี้

| ยุทธวิธีสายลับ | ทักษะ Git |
|---|---|
| แตกโลกคู่ขนานสร้างอีกตัวตน | `git switch -c <branch>` |
| กลับมาสำนึกตัวเองว่าอยู่ที่ไหน | แก้ปัญหา Detached HEAD |
| ลบหลักฐานก่อนหลุดออกไป | `git reset --hard` |
| ถอนพิษหลักฐานที่หลุดออกไปแล้ว | `git revert` |

---

## ภารกิจที่ 1: สร้างโลกคู่ขนานหลอก Interpol

### 🌐 ทฤษฎี — Branch คืออะไรในโลกสายลับ

ลองนึกภาพว่าชีวิตคุณมีสองเส้นทาง: เส้นทาง **`main`** คือตัวตนนักฆ่าที่แท้จริง และเส้นทาง **`alibi-timeline`** คือตัวตนที่คุณสร้างขึ้นมาเพื่อหลอกตำรวจ — ชายผู้ป่วยหนักนอนอยู่บนเตียงโรงพยาบาล ไม่มีทางออกไปก่อเหตุได้

Git Branch ทำงานแบบเดียวกัน: เส้นทางทั้งสองดำรงอยู่พร้อมกัน แยกออกจากกันอย่างสิ้นเชิง คุณสลับไปมาระหว่างสองตัวตนได้ตามต้องการ และเมื่อ Interpol เข้ามาตรวจ คุณก็ชี้ให้พวกเขาเห็นแค่ตัวตนที่ต้องการให้เห็น

### 🔫 ปฏิบัติการ

**ขั้นที่ 1:** สร้างโลกคู่ขนานขึ้นมา

```bash
git switch -c alibi-timeline
```

คุณเพิ่งแยกตัวเองออกจากสายหลักแล้ว ตอนนี้อยู่บนเส้นทาง `alibi-timeline`

**ขั้นที่ 2:** เปิดไฟล์ `hit_list.txt` แล้วลบรายชื่อเป้าหมายทั้งหมดออก แทนที่ด้วยตารางนัดแพทย์ปลอม เช่น:

```
ตารางนัดหมายแพทย์ — โรงพยาบาลกรุงเทพ
- 08:00  เจาะเลือดตรวจไข้หวัดใหญ่สายพันธุ์ H3N2
- 10:30  เอกซเรย์ปอด (ผลยืนยันปอดอักเสบเล็กน้อย)
- 14:00  รับน้ำเกลือและยา — นอนพักในวอร์ด C
```

**ขั้นที่ 3:** บันทึกและส่งขึ้นคลาวด์

```bash
git add hit_list.txt
git commit -m "chore: บันทึกตารางแพทย์และอาการไข้หวัดใหญ่"
git push origin alibi-timeline
```

ข้อมูลปลอมอยู่บนคลาวด์แล้ว เมื่อ Interpol ขอหมายค้นข้อมูลดิจิทัล พวกเขาจะเห็นแค่คนป่วยที่ไม่มีทางออกไปก่อเหตุได้

---

## ภารกิจที่ 2: สลับกลับโลกจริง — และออกจากกับดัก Detached HEAD

### 🌐 ทฤษฎี — Detached HEAD คืออะไร

HEAD คือ **ตำแหน่งปัจจุบันของคุณในประวัติศาสตร์** เปรียบเหมือนหมุดปักบนแผนที่ปฏิบัติการ ปกติหมุดนั้นยึดติดกับปลายสายงาน (Branch) เสมอ

แต่ถ้าคุณบังเอิญ `checkout` ไปยัง Commit เก่าโดยตรง (ไม่ใช่ชื่อ Branch) — หมุดจะหลุดออกจากสายงาน คุณลอยอยู่กลางอากาศ ไม่มีที่ยึด สถานะนั้นเรียกว่า **Detached HEAD** ทุก Commit ที่ทำในสถานะนี้จะหายไปเองเมื่อคุณสลับสายงาน เหมือนสายลับที่ทำงานนอกระบบ — ไม่มีใครรู้ ไม่มีหลักฐาน แต่ผลลัพธ์ก็หายไปพร้อมกันด้วย

### 🔫 ปฏิบัติการ

**ขั้นที่ 1:** สลับกลับสายหลัก

```bash
git switch main
```

**กับดักจากพี่ปี 2:** พี่ ๆ ผู้สอนจะแอบส่งคำสั่งให้น้องรัน ซึ่งจะทำให้น้องหลุดเข้าสถานะ Detached HEAD โดยไม่รู้ตัว สังเกตสัญญาณเตือนในเทอร์มินัล:

```
HEAD detached at a1b2c3d
```

**ขั้นที่ 2:** ถ้าพบว่าตัวเองติดอยู่ใน Detached HEAD ให้ตรวจสอบสถานะก่อน

```bash
git status
git log --oneline -5
```

**ขั้นที่ 3:** ดึงตัวเองกลับมาที่ยอดสาย main

```bash
git switch main
```

หรือถ้าต้องการเก็บงานที่ทำในสถานะ Detached HEAD ไว้ก่อน:

```bash
git switch -c escape-route
git switch main
git merge escape-route
```

คุณปลอดภัยแล้ว — กลับมาอยู่บนสายหลักเรียบร้อย

## ภารกิจที่ 3: กับดักคำสารภาพบาป และการทำลายหลักฐาน

### 🌐 ทฤษฎี — Reset vs Revert ต่างกันอย่างไร

| | `git reset --hard` | `git revert` |
|---|---|---|
| **เปรียบเหมือน** | เผาหลักฐานจนไม่เหลือซาก | ยื่นเอกสารยกเลิกอย่างเป็นทางการ |
| **ผลลัพธ์** | Commit หายไปจากประวัติ | เพิ่ม Commit ใหม่ที่ยกเลิกของเก่า |
| **ใช้เมื่อ** | ยังไม่ได้ Push ขึ้น Remote | Push ขึ้น Remote ไปแล้ว |
| **ความเสี่ยง** | ห้ามใช้หลัง Push เด็ดขาด | ปลอดภัย ทีมไม่เดือดร้อน |

### 🔫 ปฏิบัติการ — โจทย์กับดักจากพี่ปี 2

พี่ ๆ ส่งสคริปต์บรรทัดเดียวมาให้ทาง Line กลุ่มลับ อ้างว่าเป็น "คำสั่งอัปเดตระบบรักษาความปลอดภัย" ให้ก๊อปวางใน Terminal แล้วกด Enter

พอรันปุ๊บ — สคริปต์นั้นแอบเขียนข้อความลงในไฟล์แล้ว Commit อัตโนมัติทันที:

```
confession.txt:
"ผมใช้ปืนสไนเปอร์แบรนด์ยุโรปส่องเข้าที่ขมับขวา เวลา 23:47 น. วันที่ 12 มิถุนายน"
```

ตอนนี้ใน `git log` มีหลักฐานมัดตัวคุณโผล่ขึ้นมาแล้ว

---

### มาตรการที่ 1: ลบร่องรอยก่อนหลุดออก (ยังไม่ได้ Push)

ตรวจสอบ `git log` ก่อนว่า Commit นั้นยังอยู่แค่ในเครื่อง:

```bash
git log --oneline -5
```

จดจำ hash ของ Commit ที่อยู่ **ก่อนหน้า** Commit สารภาพบาป แล้วทุบทิ้ง:

```bash
git reset --hard <commit-hash-ก่อนหน้า>
```

Commit สารภาพบาปหายไปจากโลกนี้แล้ว ไม่มีร่องรอย Interpol ไม่มีวันพิสูจน์ได้

---

### มาตรการที่ 2: ถอนพิษหลักฐานที่หลุดออกไปแล้ว (Push ไปแล้ว)

สมมติว่าสายลับมือใหม่กด Push ไปก่อนจะทันสังเกต ห้ามใช้ `reset` เด็ดขาด เพราะประวัติของทุกคนในทีมจะพัง

วิธีที่ถูกต้องคือสร้าง Commit ใหม่ที่ **ยกเลิกผลของ Commit เก่า** อย่างเป็นระเบียบ:

```bash
git log --oneline -5
# หา hash ของ Commit สารภาพบาป

git revert <commit-hash-สารภาพบาป>
# Editor จะเปิดขึ้นมาให้แก้ข้อความ Commit
# บันทึกแล้วปิด

git push origin main
```

ใน `git log` จะมีสองบรรทัด:

```
a9f3c1e  Revert "confession: บันทึกการปฏิบัติการ"   ← ตัวถอนพิษ
b7e2d4a  confession: บันทึกการปฏิบัติการ             ← ของเก่าที่ถูกยกเลิกแล้ว
```

ประวัติยังอยู่ครบ แต่ผลลัพธ์สุดท้ายสะอาด — เหมือนสายลับที่ยอมรับความผิดพลาดอย่างเป็นทางการ แล้วแก้ไขมันอย่างมีระเบียบวินัย

---

## 📋 สรุปปฏิบัติการ Class 3

```
✅ สร้างโลกคู่ขนาน      → git switch -c alibi-timeline
✅ Push ข้อมูลลวงตา     → git push origin alibi-timeline
✅ กลับสายหลัก          → git switch main
✅ แก้ปม Detached HEAD  → git switch main / git switch -c <ชื่อใหม่>
✅ ทุบหลักฐาน (Local)   → git reset --hard <hash>
✅ ถอนพิษ (Remote)      → git revert <hash>
```

---

## ⚠️ กฎเหล็กของ Sicario-script

> **ห้ามใช้ `git reset --hard` กับ Commit ที่ Push ขึ้น Remote ไปแล้วเด็ดขาด**  
> ประวัติของทุกคนในทีมจะกระจัดกระจาย สมาชิกคนอื่นจะสับสน และ Interpol จะรู้ทันทีว่ามีคนพยายามซ่อนอะไรบางอย่าง  
> ทางรอดเดียวในสถานการณ์นั้นคือ `git revert` — สุภาพ เป็นระเบียบ และทีมไม่เดือดร้อน
